Lesson 7

It is possible to define a script in your node application by adding a “script” section to your “package.json”. You can then run your script by navigating to the correct directory and typing “npm run “script-name” in the console.
To define an environment variable in node you can enter it by writing “Variable = value” followed by the by “node app.js“ to start with the given value as an environment variable. This doesn't work with windows though, and instead you need to write it like: “$env:Variable="value". An alternative to this is to install the package cross-env which allows windows users to define the environment variables the usual way.
Rendering can happen both on the client side and server side. It can be preferable to render on the server side since you can’t always rely on the client having the resources available to render quickly, thereby improving customer satisfaction. However with mobile apps it is very common to have client-side rendering since modern phones have a lot of processing power nowadays. It’s also more resource intensive to have SSR as it uses your own resources contrary to client-side rendering. Server side rendering means that the HTML page is rendered on the server before it is sent to the client. SSR can also be important for search engine optimization as the crawlers don’t usually load javascript, so websites with SSR get a higher score.
This also implies that depending on where code is it uses resources from either client or server. If a fetch request is located in your backend code like app.js it uses server resources, but if it is in the frontend pages in our public folder it uses the clients resources.
